# 001 — Три таба вместо двух

Статус: принято
Фаза: 2
Влияет на: фазы 3-7 (все экраны привязаны к табам)

## Контекст

В шаблоне было 2 таба (Home, Settings). Для приложения учёта смен нужна другая структура навигации, отражающая основные сценарии использования.

## Решение

Три таба: **Смена** (трекер), **История** (список + статистика), **Профиль** (данные + организации).

- Смена — основной рабочий экран, должен быть первым
- История — часто используется для проверки отработанного времени
- Профиль — редко используется, но содержит важные настройки и вход в организации

Организации доступны из Профиля через nested navigation, а не как отдельный таб, т.к. не все пользователи состоят в организациях (персональный режим).

## Последствия

- MainRouterPage нужно переделать с 2 табов на 3
- Все экраны организаций — nested navigation из профиля
- Экран трекера смены должен поддерживать оба режима (персональный + организационный) на одном экране
